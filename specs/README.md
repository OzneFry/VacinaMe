# Specs do projeto

Esta pasta centraliza as especificações de features de forma simples e incremental.

## Objetivo

Usar Spec-Driven Development (SDD) de maneira leve, sem introduzir arquitetura nova ou complexidade extra.

## Como usar

1. Antes de implementar uma nova feature, crie uma spec em Markdown.
2. Use o template padrão como ponto de partida.
3. Implemente a feature a partir da spec.
4. Atualize a spec se a solução exigir mudanças de comportamento.

## Estrutura

- template.md: modelo base para novas specs
- examples/: exemplos reais de specs já usadas no projeto

## Regra prática

Se a feature for pequena, a spec pode ser curta. Se for maior, detalhe regras, fluxos e critérios de aceite.
